Terminologi Git


Repository

Ett repository är en plats som innehåller lagrad information och data om användare, människor, datorer eller annan väsentlig information.


Branch

En branch, gren, representerar en självständig linje av utveckling. Git branch-kommando kan skapa, lista, döpa om eller ta bort branches. Jag kan inte hoppa mellan grenar, eller återskapa historian tillhörande en fork. På grund av detta är Git branch strikt integrerad med kommandon Git checkout och Git merge.


Clone

En Git clone är ett Git command line verktyg som används för att hitta ett existerande repository och skapa en klon eller kopia av sagda repository.


Commit

Commit är ett kommando som fångar en ögonblicksbild av ett projekts väntande ändringar. Committade snapshots kan liknas vid säkra versioner, då Git aldrig kommer ändra dem om jag inte explicit ber om det.


Push

Git push används för att ladda upp innehåll från ett lokalt repository till ett remote repository. Genom att pusha mina commits, överför jag mina lokala ändringar till repot som ligger på en server. Det är motsatsen till Git fetch, där fetch importerar commits till lokala branches, medans push exporterar commits till remote repo.


Pull

Git pull används för att hämta och ladda ner innehåll från ett remote repository och ögonblickligen uppdatera det lokala repot för att matcha det innehållet. Att merge uppströms ändringar till mitt lokala rep är en vanlig uppgift i Git-baserade arbetssätt.


Merge

Git merge kombinerar sekvenser av commits till en uniform historia. Vanligast används git merge för att kombinera två branches. När Git hittar ett gemensamt bas-commit, skapar den ett nytt merge-commit som kombinerar ändringarna i varje köad merge-commit sekvens.


Working directory

Working directory är en enskild titt på en version av projektet. Detta betyder att om jag går till (checkout) en branch (tex master) och är i ett specifikt commit (tex HEAD), är working directory 'paraply' mappen för alla mina filer och mappar.


Staging

Att stagea en fil är att förbereda den för en commit. Git och dess index tillåter mig att commita vissa delar av ändringar jag har gjort sedan senaste commit. Detta kan vara bra utifall en jobbar på två funktioner, där den ena är klar och den andra behöver mer jobb.


HEAD

Termen HEAD syftar på det särskilda commit som jag tittar på. Standard är att jag kommer att se toppen på master branch i ett repository, såvida inte master branch har ett annat namn. Toppen på master branch är den senaste commiten till main branch i min codebase.


Revert

Git revert används för att backa ändringar i ett repositorys historia. Andra 'undo' kommandon såsom Git checkout och Git reset, flyttar HEAD och referens punkter till ett speciellt commit. Git revert tar ett speciellt commit men flyttar inga referens punkter till detta commit.


Reset

Git reset är ett kraftfullt kommando som används för att backa lokala ändringar till repo. Git reset utgår från 'The Three Trees of Git'. Dessa träd är Commit History (HEAD), the Staging Index och Working Directory. Det finns tre alternativ i command line som utgår från dessa träd.


Fork

Fork är när jag gör en kopia av ett repo med syftet att utveckla projektet på egen hand, utan att dela min vidareutveckling med original repo. Skillnaden mellan fork och clone är att med en fork är syftet inte att jobba vidare med originalet; det skall förbli orört. Branch är annorlunda från fork därför att en branch utgår från ett repository, medan en fork är ett helt repository.
