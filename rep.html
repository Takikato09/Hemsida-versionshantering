Repository

Skillnaden mellan ett lokalt repository och ett remote repository är följande:
Ett lokalt är det du har på din dator medan remote är något som ligger på en remote dator. Syftet med detta är att Git är ett distribuerat versionskontrollsystem och det gör Git fantastiskt. Ditt lokala förråd har exakt samma funktioner och funktionalitet som alla andra Git-förvar. Så ett Git-repo på en server är detsamma som ett Git-repo på GitHub (beviljat GitHub lägger till ytterligare funktioner, men i dess kärna har du att göra med Git-repositories) vilket är samma som din kollegas lokala repo.

Så varför är det fantastiskt? Eftersom det inte finns någon central repo måste du ha tillgång för att utföra ditt arbete. Du kan begå, förgrena och festa på ditt eget repo på din lokala dator även utan internetåtkomst. Sedan, när du har en anslutning igen, kan du skicka dina ändringar till alla andra Git-repo du har tillgång till. Så medan de flesta människor behandlar en viss repo som den centrala repan (förvaret), är det ett processval, inte ett Git-krav.

Poängen med allt det var att säga (som andra har) att du överlåter din README till din lokala repo. Sedan, när du vill, kan du överföra ändringar från din lokala repo till vilken annan repo som helst. Det är ganska snyggt!
